JavaScript中的类型转换（上）

2019年 四月 02

这是介绍 JavaScript 中类型转换的连续三篇 Blog 中的第一篇，希望对读者在这个问题的理解上有帮助，如果你需要更详尽的阐述或说明，请阅读 Kyle Simpson 的《你不知道的 JavaScript》。

JavaScript 中的类型转换分为两种类型，一种是显性的类型转换(type casting / explicit coercion)，另外一种是隐性的类型转换(coercion / implicit coercion)。在这两种类型转换中，显性转换比较好识别和理解，比如 String(32)，Boolean(0)，在看到这样的代码的时候，开发者可以很清晰直观的辨别出来类型转换的操作。相比之下隐性的转换就晦涩很多。显式和隐式是相对的概念，对一个 JavaScript 专家，甚至是 JS 标准的制定者，可能所有的隐性转换在他们看来都是显式的转换，因为他们掌握了所有的 JavaScript 细节，在他们看来一切都是清晰明了的。但是我们大多数人都不是专家或者标准制定者，这个文章的目标读者是对 JavaScript 有良好理解的普通程序员。还有一点要明确的是，大多数情况下，程序员写的代码都会有其它人来阅读或修改，即使你能理解所有不同的类型转换，想想你们团队中的新成员，想想将来维护你的代码的人，有可能他们很难理解你代码中出现的隐性类型转换，这一点必须要考虑到。

值的抽象操作

JavaScript 中类型转换的结果只可能是String、Number、Boolean三个类型中的一种。不同的类型如何转换为这三种类型的值，ES5 标准定义了几种抽象操作 ToString、ToNumber，ToBoolean 和 ToPrimitive。抽象操作是 ES5 标准中“仅内部操作”的另一种说法。任何非 String 类型的值转换为 String 时，转换的过程由ToString 抽象操作来处理。同样的原理作用于ToNumber和ToBoolean。

  • ToString

    每一种原始值，标准中给出了 ToString 操作的结果。null 的结果是null，undefined 的结果是undefined，布尔 true 的结果是true。数值类型的值会被转换为对应的字符串的结果，太大或太小的数字会用科学计数法来表示。

    普通的对象，默认的 toString()方法会返回对象的内部的[[Class]]，比如[object Object]。默认的 toString 方法在原型链Object.prototype.toString()上，可以用自定义方法覆盖这个方法。这样当对象需要转换成 String 类型的时候，自定义的 toString 方法就会生效。

    String({}) //输出结果 '[object Object]'

String({
toString() {
  return 'abc'
},
}) //输出结果 'abc'

    Array 对象上的 toString 方法覆盖了Object.prototype上的默认方法。把 Array 转换为 String 的结果是，一个包含所有数组元素的字符串，每个元素之间用逗号分隔。

    const a = [1, 2, 3]

String(a) // 输出结果 '1, 2, 3'
a.toString() // 输出结果 '1, 2, 3'

    toString()方法既可以直接调用，也可以在需要将 Array 转换为 String 的地方自动调用。

    JSON.stringify()工具方法可以将目标参数转换成 JSON 兼容的字符串形式。转换的规则和ToString有相同和不同的地方。对于简单的值，stringify和ToString的结果是一样的。一些值被认为不兼容 JSON，这样的值有undefined、函数、ES6 的Symbol、有循环索引的对象(比如：const a = { b: a})。标准的 JSON 结构认为这样的值是无效的，主要原因是其它使用 JSON 数据的语言无法使用这些值。JSON.stringify()方法转换时遇到这些值时，会自动忽略掉它们。如果这样的值在Array中出现，会被替换为null，这样做的目的是不改变数据的长度和元素的位置。如果这样的值出现在 object 的属性中，这条属性会被排除掉。

    JSON.stringify(undefined) // 输出结果 undefined

JSON.stringify(function() {}) // 输出结果 undefined

JSON.stringify([1, undefined, function() {}, 4]) // 输出结果 "[1, null, null, 4]"

JSON.stringify({ a: 2, b: function() {} }) // 输出结果 "{"a": 2}"

    如果使用JSON.stringify()方法去 Json 化一个有循环索引的对象，会抛出一个错误。

    可以定义toJSON()方法去影响对象的 Json 化结果。如果一个对象上有toJSON方法的话，JSON.stringify()会先调用这个方法，这样可以解决前面提到的一些问题。

    const o = {}

const a = { b: 42, c: 0, d: function() {} }
// 创建一个循环引用
o.e = a

a.toJSON = function() {
return { b: this.b }
}

JSON.stringify(a) // 输出结果 "{"b": 42}"

    注意toJSON()方法返回的是兼容 JSON 的值，用于进行 JSON 字符化，而不是 JSON 字符化的结果。

    JSON.stringify()不是类型转换的一种形式，但它和ToString抽象操作有一些两个相似的地方。

    1. string、number、boolean和null转换成 JSON 的规则和转换成string的规则是一样的。
    2. 如果被转化为 Json 的对象上有toJSON方法，JSON.stringify会调用这个方法。类似对象被转换成 string 时，会调用toString()方法。

  • ToNumber

    对于此操作，布尔值true结果是1、false结果是0、undefined结果是NaN、null的结果是0。string类型的值转换成数值类型时会尝试解析该字符串为数字，如果解析失败会返回NaN。对象转换为 number 时会先尝试将该对象转换为原始值，如果这个原始值不是数字类型，再依照上面的规则转换。对象转换为原始值用的是对象上的valueOf()和toString()方法，ToPrimitive操作按顺序尝试这两个方法，如果这两个方法都不能返回一个原始值，会抛出一个TypeError错误。

  • ToBoolean

    JavaScript 有true和false关键字，用来表示逻辑值真和假。注意，在其它语言中1和0等同于true和false，但是在 JavaScript 中他们是不一样的类型，只是互相可以转换。

    JavaScript 标准规定了会转换为false的值，这些值如下:

    • undefined
    • null
    • false
    • +0, -0, NaN
    • ""

    不在这个列表中的其它值在转换为布尔值时都会转换为true。
