去除inline元素之间空白的N种方法

2019年 四月 03

几个 inline 元素排列在一起的时候，它们之间会留有空白，这些空白往往不是我们想要的结果，今天我们来看看几个去掉 inline 元素留白的方法。

假设我们要写一个水平方向的菜单，每个菜单宽度一样，占满屏幕的宽度。我打算用ul和数个li元素实现这个目的，html 和 CSS 的代码如下:

<ul class="menu">
  <li class="menu__item">菜单1</li>
  <li class="menu__item">菜单2</li>
  <li class="menu__item">菜单3</li>
  <li class="menu__item">菜单4</li>
  <li class="menu__item">菜单5</li>
</ul>
*,
*::before,
*::after {
  padding: 0;
  margin: 0;
  box-sizing: inherit;
}

:root {
  font-size: 62.5%;
  box-sizing: border-box;
}

.menu {
  list-style: none;
}

.menu__item {
  font-size: 2rem;
  display: inline-block;
  width: 20%;
  background-color: lightcoral;
}

结果并不是我希望的目录刚好占一行，并且每个元素的宽度是百分之 25，但是每个目录中间有一点空白区域，正是因为这些空白区域，最后一个菜单因为宽度不够，自动移到了下一行。

使用负的外边距
.menu__item {
  margin-right: -2.6px;
}

在我的例子中，使用-2.6px 刚好可以抵消 inline 元素的空白，在实际使用中这个数值需要根据父元素上的字体大小调整。这个方法在 IE6、7 上有 Bug，如果你不需要考虑这两个浏览器的表现的话，用负外边距是一个很好的方法。

忽略元素闭合标签

使用 HTML5 的时候，你还可以忽略元素闭合标签，也能达到消除空白的目的。

当然这样写和第一种方法一样怪异，故意省略闭合标签？不推荐这种方法。

设父元素的字体大小为 0
.menu {
  font-size: 0;
}

.menu__item {
  font-size: 2rem;
}

这种方法优点是，不用做额外的 css 数值计算，也不用写奇怪的 html 代码。缺点是如果子元素上的字体是用em，从而依赖于父元素的字体大小，用这种方法元素上的字体大小就会被设为零。解决的办法是用rem，或是其它不依赖父元素字体尺寸的单位。

最后总结，其实在碰到需要消除空白的这种情况，首先要考虑是不是真的需要使用inline或inline-block。大多数情况下，使用float和flexbox是更合适的选择。
